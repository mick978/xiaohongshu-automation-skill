# 实拍照片 → 爆款文案 → 发布 (2026-07-03 实战)

## 场景
user提供一组实拍照片（`/Photo/*.jpeg`，5 张高层阳台日落晚霞），要求写小红书爆款文案并发布。与 `viral-caption-patterns.md` 的区别：后者是从已有笔记提取文案套路，本流程是**从用户自己的照片提取视觉信息→写文案→发布**。

## 完整 SOP（5 步）

### Step 1: 列出照片
```bash
ls -la /Photo/  # 或用户指定目录
```
确认照片数量、格式、大小。

### Step 2: 逐张 vision_analyze
对每张照片调用 `vision_analyze`，question 用：
> "详细描述这张照片的内容、场景、地点、人物、活动、色调、氛围。这是小红书笔记的素材图。"

**关键**：必须逐张看，不能跳过。每张照片可能有独特细节（不同天色/角度/时间段）。

### Step 3: 提取共性 + 差异 → 构思文案

从所有 vision_analyze 结果中提取：

| 维度 | 提取什么 | 示例（本 session） |
|------|---------|-------------------|
| **共性** | 所有照片的共同元素 | 同一高层窗口、城市日落、火烧云、远山剪影、住宅楼群 |
| **差异** | 每张照片的独特之处 | 不同天色（橘红/粉紫/灰蓝）、不同时间段、云层形态变化 |
| **色调** | 主色调 + 渐变层次 | 橘红→粉紫→灰蓝渐变、暖冷对比 |
| **氛围** | 情绪关键词 | 治愈、宁静、烟火气、打工人下班 |
| **地点线索** | 建筑/地形特征推测地点 | 南方城市丘陵、高层住宅、远山=龙泉山 |

### Step 4: 套爆款模板写文案

结合 `viral-caption-patterns.md` 的 6 套路：

**本 session 实例**（5 张日落照 → "贩卖黄昏"赛道）：

```
标题：住30楼的好处｜每天下班都能收割一片橘子海🌇

正文结构：
① 反差钩子 → "都说成都看不到好日落，那是你没住够高"
② 场景描述 → 引用具体视觉细节（5月火烧云/橘红粉紫鸭蛋蓝/龙泉山剪影/万家灯火）
③ 情绪价值 → "不花钱 不用跑，推开门就是免费 IMAX"
④ 时间跨度 → "这些照片拍了整整一个月，同一个窗口不同天色"
⑤ 金句收尾 → "你今天辛苦了，到家了就好"
⑥ 实用信息 → 📍定位 + 🕥最佳拍摄时间
⑦ 话题标签 → #成都日落 #晚霞 #治愈系天空 #贩卖黄昏 #住高层的快乐 #下班日常 #天空收藏家
```

**文案质量要点**：
- 引用 vision_analyze 看到的**具体视觉细节**（颜色/建筑/山名/时间段），不是空泛形容
- 标题用反差+数字+emoji（"住30楼的好处"= 反差 + "橘子海"= 视觉钩子）
- 正文有情绪转折（"都说...那是你没..."）
- 金句必须有，放文末（"你今天辛苦了，到家了就好"）
- 标签 8-12 个，覆盖赛道词 + 情绪词 + 地域词

### Step 5: 发布

```bash
xhs post \
  --title "标题" \
  --body "正文（多行，含 emoji 和 #标签）" \
  --images "/path/to/img1.jpeg" \
  --images "/path/to/img2.jpeg" \
  --images "/path/to/img3.jpeg" \
  --images "/path/to/img4.jpeg" \
  --images "/path/to/img5.jpeg" \
  --topic "晚霞" \
  --json
```

**关键 pitfall（本 session 踩过）**：
- `--images` 必须**多次 flag**，逗号分隔会报错（FileNotFoundError）
- `--body` 可以多行（用 `\n` 或 shell heredoc）
- `--topic` 会自动搜索匹配已有话题并附加
- 返回 `{"ok": true, "data": {"id": "note_id", "score": 10}}` = 发布成功

## 发布后验证

```bash
xhs my-notes --json  # 查看刚发的笔记
```

## ao 专家文案路径 (2026-07-03 验证)

user对文案要求 = "爆款级"，agent 自写文案**会被否决**（"不够爆款"）。必须走 `ao compose` 5 步专家工作流。

### 触发条件
- user说"文案不够爆款"/"重新设计文案"/"调用专家" → 走 ao
- user否决 agent 自写文案 → 走 ao
- 默认：**新笔记发布前一律走 ao**，不再自写

### ao compose prompt 模板

```bash
ao compose "你是小红书爆款文案专家。我有一组[照片主题]照片（[张数]张精选，[拍摄方式]，含[具体画面描述]）。当前标题'[旧标题]'不够爆款。请从以下维度重新设计：1) 提供10个备选标题，按爆款潜力排序，说明每个标题的钩子逻辑 2) 重新撰写正文文案（要求：前3行必须有强情绪钩子、中间用短句节奏感、结尾有互动引导CTA、emoji克制不泛滥、字数控制在300字内）3) 推荐5个核心话题标签。目标受众：[受众描述]，情绪共鸣点='[核心情绪]'。参考爆款模式：数字悬念型、反差冲突型、情绪共鸣型、身份认同型。" --run --provider claude-code 2>&1
```

### ao 输出结构（5 步，~267s，34k tokens）

| 步骤 | 角色 | 产出 |
|---|---|---|
| 1 | 📕 小红书策略师 | 爆款模式匹配 + 情绪关键词 + 标题方向 |
| 2 | 🎣 标题爆破师 | 10 个备选标题按潜力排序 + 钩子逻辑 |
| 3 | ✍️ 文案操盘手 | 三段式正文（下沉→上扬→留白） |
| 4 | 🏷️ 标签策略师 | 5 标签金字塔排列 + 流量类型标注 |
| 5 | 📋 定稿主编 | TOP 3 标题 + 完整文案 + 标签整合 |

### 选用规则
- **默认选 TOP 1 标题** + 定稿正文 + 标签策略师的 5 标签
- TOP 2/3 留作 ABT 测试备选（同图不同标题再发）
- ao 的标签排序有算法逻辑（前 3 个权重最高），**按 ao 排序用，不要自己改序**

### 用 ao 文案重新发布

```bash
# body 写入临时文件避免 shell 转义问题
echo "ao 定稿正文" > /tmp/xhs_body.txt
xhs post \
  --title "ao TOP1 标题" \
  --body "$(cat /tmp/xhs_body.txt)" \
  --images img1.jpeg --images img2.jpeg --images img3.jpeg \
  --images img4.jpeg --images img5.jpeg \
  --topic "标签1" --topic "标签2" --topic "标签3" --topic "标签4" --topic "标签5" \
  --json
```

### 实战数据（2026-07-03）

| 版本 | 标题 | note_id | 来源 |
|---|---|---|---|
| v1 自写 | 住30楼的好处｜每天下班都能收割一片橘子海 | 6a47226f... | agent 自写 |
| v2 自写 | 同一个窗口拍了30天天空｜原来成都这么浪漫 | 6a47230a... | agent 自写，user否决"不够爆款" |
| v3 ao 专家 | 被困工位的第30天，我偷了一整片黄昏 | 6a4725b4... | ao 5步工作流 |

**教训**：v1/v2 自写均不够爆款，ao 出的"反差冲突型"标题（前8字锁定痛点+"偷"字撑开反差）明显更强。

### 批量照片 vision_analyze 筛选 (14+ 张场景)

当照片数量 ≥10 张时，不需要全发，用 vision_analyze 逐张打分筛选 top 5：

1. 对每张照片调 `vision_analyze`，question 用："一句话总结这张照片的核心画面和色调。评分1-10，适合做小红书封面吗？"
2. 汇总评分表，按分数排序
3. 选 top 5 发布（封面用最高分的那张）
4. 评分维度：视觉冲击力 / 色彩层次 / 构图 / 情绪氛围 / 封面缩略图抓眼球程度

**实战**：19 张照片 → 14 张新增逐张 vision → top 5（9/10 + 3×8.5/10）→ 封面用 9 分那张

## 与 viral-caption-patterns.md 的关系

- `viral-caption-patterns.md`：从**已有爆款笔记**提取文案套路（6 套路 + 3 模板），用于**研究后仿写**
- 本文件：从**用户实拍照片**提取视觉信息→写文案→发布，用于**照片驱动创作**
- 两者互补：先看照片定主题 → 套 viral-caption-patterns 的模板写正文
- **ao 专家路径**：当 agent 自写文案不够爆款时，走 `ao compose` 5 步工作流。详见上方"ao 专家文案路径"节
