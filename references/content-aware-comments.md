# 内容感知评论生成算法 (2026-06-14)

## 问题
用户明确投诉"评论牛头不对马嘴"，后要求"中性评论，不要夸张"。

## 方案：三维提取 + 已有评论分析 + 中性风格

### 第一步：读已有评论，分析避免重复

在生成评论前，先读该笔记已有的评论（取前 8 条），分析：
- 别人是否已经说过"好看/好美/漂亮" → 避免重复夸
- 别人是否已经说过"收藏" → 避免再说"收藏了"
- 别人是否已经说过"去过" → 避免再说"去过"

```python
existing_comments = []
comments_data = xhs('comments', note_id, '--xsec-token', token)
for c in comments_data['data']['comments']:
    existing_comments.append(c['content'][:100])

existing_text = ' '.join(existing_comments).lower()
said_scenic = any(w in existing_text for w in ['好看', '好美', '漂亮', '绝了'])
said_collect = '收藏' in existing_text
```

### 第二步：从标题+正文提取三个维度

1. **地点** — 龙泉山、蟠龙谷、青城山、天府绿道等 30+ 成都周边地名
2. **活动** — 徒步、骑行、攀岩、装备、穿搭、避暑、亲子等 20+ 活动类型
3. **特征** — 免费、公交可达、小众、新手友好、22℃等 15+ 特征词

### 第三步：标题关键词优先

标题中的活动词优先级 > 正文中的。避免"攀岩"笔记被"徒步"抢了匹配。

```python
# 标题优先
title_lower = title.lower()
for kw, name in act_map.items():
    if kw in title_lower:
        activities.append(name)
# 再补充正文
for kw, name in act_map.items():
    if kw in combined and name not in activities:
        activities.append(name)
```

### 第四步：地点×活动 组合生成中性评论

**中性风格原则：像真人随口一句，不是营销号**

| 地点 | 活动 | ❌ 太夸张 | ✅ 中性 |
|------|------|----------|--------|
| 蟠龙谷 | 避暑 | "蟠龙谷避暑绝了！光看温度就想去🧊" | "蟠龙谷这个温度确实舒服，夏天去刚好" |
| 龙泉山 | 徒步 | "龙泉山看着就好舒服！收藏了找时间去🌿" | "龙泉山这个收藏了，找时间去" |
| 天府绿道 | 骑行 | "在天府绿道骑车太舒服了！风景好路也平" | "天府绿道路况看着不错，找个时间骑一圈" |
| (无) | 攀岩 | "攀岩看着好有意思！一直想试试" | "攀岩看着挺有意思，这个难度新手能试吗" |
| (无) | 装备 | "装备清单太实用了！正好在纠结买什么📋" | "装备推荐挺实用的，参考了" |

**中性风格特征：**
- 不用感叹号（或最多 1 个）
- 不堆 emoji（0-1 个）
- 可以提问（"这个难度新手能试吗"）
- 可以说"参考了"、"先记下"这种真实反应
- 不说"绝了"、"太强了"、"必须去"这种夸张词

### 第五步：兜底逻辑
- 有地点无活动 → "XX这个收藏了，找时间去" / "XX看着不错，周末可以安排"
- 有活动无地点 → 按活动类型选中性通用评论
- 有特征无地点无活动 → 按特征匹配（"免费+公交直达这种最实用"）
- 全无 → "成都周边好地方真多，先记下了" / "感谢分享，成都周末又有去处了"

## 禁止的做法
- ❌ 随机模板评论（"好看"、"666"、"已收藏"）
- ❌ 不读内容直接评论
- ❌ 通用评论用于所有笔记
- ❌ 评论内容与笔记主题完全无关
- ❌ 夸张语气（"绝了！"、"太强了！"、"必须去！"）
- ❌ emoji 轰炸（一个评论 3+ 个 emoji）
- ❌ 不看别人评论就重复同样的观点

## 升级：全文 + 图片感知评论 (2026-07-02 实战验证)

**背景**: 三维提取算法仍可能"牛头不对马嘴"——标题只提到"好舒服的青城山"，但正文写了"后山大环线+瀑布+松鼠+下雨+没蛇"。仅靠标题关键词会漏掉正文关键信息，导致评论泛泛。

**用户原话纠正 (2026-07-02)**: "评论请看好标题和文案内容。识别图片内容，在精心构思进行评论。不要太随意，牛头不对马嘴"

### 新工作流：read 全文 + vision_analyze 封面图 → 精写评论

```
Step 1: xhs read <id> --xsec-token <tok> --json
        → 提取 note_card.desc（完整正文）、title、tag_list

Step 2: vision_analyze(cover_url, "描述图片中的场景、人物、氛围")
        → 拿到图片内容描述（场景/光线/人物动作/氛围）
        → 部分图片可能 403（防盗链），跳过 vision 只用文本

Step 3: 综合 desc + 图片描述，精写评论
        → 评论必须引用笔记中的【具体细节】
        → 示例：desc 提到"骑累了钻进路边咖啡店" → 评论"骑累了钻进路边咖啡店这个安排太对了"
        → 示例：desc 提到"后山大环线+瀑布+松鼠" → 评论"一路有瀑布和松鼠这也太舒服了"
        → 示例：图片显示"SPECIALIZED 公路车+锦城湖+夕阳" → 评论"锦城湖+夕阳这个机位绝了！SPECIALIZED 配白色骑行服也太飒了"

Step 4: 每条评论必须唯一，禁止跨笔记复用
```

### 关键教训
- **标题≠内容**：标题"好舒服的青城山"没说后山大环线/瀑布/松鼠，必须读 desc
- **图片有信息量**：封面图的品牌、地点、光线、天气都是评论素材
- **反讽/幽默文案要接梗**：张蘑菇"有什么好炫耀的"是反讽，评论要接梗不是当真
- **攻略型笔记要夸结构**：双环线/公共交通/避雷tips 这种攻略结构值得夸具体
- **vision_analyze 可能 403**：部分封面图防盗链，跳过即可，只用文本也能写出好评论
